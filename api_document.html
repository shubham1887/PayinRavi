Prepaid Recharge & Special & DTH & Postpaid Bill

S.No Attribute Descriptions Required
1 userid userid (provided by Website) true
2 pin pin (provided by Website) true
3 number Recharge Mobile Number true
4 operatorcode operatorcode (see servicename list at last) true
5 amount Recharge Amount true
5 uniqueid Client unique Id true
6 circlecode * true
GET: http://payin.live/API/recharge?userid=UserId&pin=pin&number=MobileNo&operatorcode=OpCode&amount=Amount&uniqueid=UniqueID&circlecode=* 
    
	
	        {"status":"Success","tid":234234,"order_id":"234234","mobile":"9898098980","amount":100,"operator_id":"RM2342344234"}
       
        
       

        status : Success / Failure / Pending
        tid : our web ref Id
        order_id : your Web Ref Id
        mobile : recharge mobile number
        amount : denomination
        operator_id : operator transaction id
		Note : We Dont Accept  Transaction with Same UniqueId.

Check Transaction Status

S.No Attribute Descriptions Required
1 username UserId (provided by Website) true
2 pwd pin (provided by Website) true
3 order_id Client unique Id true
GET: http://payin.live/API/status/statuswithopid?username=UserId&pwd=pin&order_id=UniqueID 
    
        Success#RM234234234234#Number=8982349834

        Status : Success / Failure / Pending
    
	Note : While StatusCheck You got Pending In Response 
		Then You have to Check Status Again For That Transaction.
	FailCase : if Transaction goes Pending To Fail Then we Automatically Refund Amount 
			in Your Account and Transaction Status goes in Failure at website .

CallBack Response

S.No Attribute Descriptions Required
1 status Success / Failure true
2 operator_id (MH02565853234)Operator Transaction ID true
3 uniqueid (8035)Your Unique TransactionID true
4 transaction_id (256)Our System TransactionID optional
5 number (7990607312)Recharge MobileNo true
6 amount (30)Recharge Amount true
GET: http://yourdomainname/Rechargeapi.aspx?uniqueid=#uniqueid#&status=#status#&operator_id=#operator_id#&transaction_id=#transaction_id#&number=#Mob#&amount=#Amt# 
    
Note : 
1)We Send CallBack Response In GET Format Only.
2)if you can't Get CallBack Response In your Website.
3)implement Status Check Url In Your Panel For Transaction Status
4)We Only Give CallBack Response For Status Pending To Success,Failure
5)We Can't Give CallBack Response For Direct Success,Failure Response.

Example :
http://yourdomainname/Rechargeapi.aspx?uniqueid=123123&status=Success&operator_id=UW234234234&transaction_id=38394&number=9898343434&amount=35
